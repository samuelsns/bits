Introduction to Testing Metrics

Testing metrics are quantitative measurements that help evaluate the effectiveness and efficiency of testing activities. They provide insights into the quality of the software, the progress of testing, and areas that need improvement.

Why Measure Testing?

  • Visibility - Provide transparency into testing activities
  • Decision Making - Support data-driven decisions about quality
  • Process Improvement - Identify areas for improvement
  • Resource Allocation - Optimize testing resources
  • Predictability - Forecast testing effort and completion
  • Communication - Facilitate discussions with stakeholders

Key Testing Metrics and KPIs

Defect Metrics

  • Defect Density - Number of defects per size of the software (e.g., per KLOC or function point)
  • Defect Distribution - Categorization of defects by type, severity, priority, etc.
  • Defect Removal Efficiency (DRE) - Percentage of defects found before release
  • Defect Leakage - Percentage of defects that escaped to production
  • Defect Age - Time between defect detection and resolution
  • Defect Trend - Pattern of defect discovery over time
Example: Calculating Defect Metrics
// JavaScript function to calculate defect metrics
function calculateDefectMetrics(defects, codeSize, releasedDefects) {
  // Defect Density = Total Defects / Code Size
  const defectDensity = defects.length / codeSize;
  
  // Defect Distribution by Severity
  const severityDistribution = defects.reduce((acc, defect) => {
    acc[defect.severity] = (acc[defect.severity] || 0) + 1;
    return acc;
  }, {});
  
  // Defect Removal Efficiency (DRE)
  // DRE = (Defects found before release / Total defects) * 100
  const totalDefects = defects.length + releasedDefects.length;
  const dre = (defects.length / totalDefects) * 100;
  
  // Defect Leakage
  // Leakage = (Defects found after release / Total defects) * 100
  const leakage = (releasedDefects.length / totalDefects) * 100;
  
  // Average Defect Age (in days)
  const totalAge = defects.reduce((sum, defect) => {
    const detectionDate = new Date(defect.detectionDate);
    const resolutionDate = new Date(defect.resolutionDate);
    const ageInDays = (resolutionDate - detectionDate) / (1000 * 60 * 60 * 24);
    return sum + ageInDays;
  }, 0);
  const avgDefectAge = totalAge / defects.length;
  
  return {
    defectDensity,
    severityDistribution,
    dre,
    leakage,
    avgDefectAge
  };
}

  { id: 1, severity: 'Critical', detectionDate: '2023-01-01', resolutionDate: '2023-01-03' },
  { id: 2, severity: 'Major', detectionDate: '2023-01-02', resolutionDate: '2023-01-05' },
  { id: 3, severity: 'Minor', detectionDate: '2023-01-03', resolutionDate: '2023-01-04' },
  { id: 4, severity: 'Critical', detectionDate: '2023-01-05', resolutionDate: '2023-01-08' },
  { id: 5, severity: 'Major', detectionDate: '2023-01-06', resolutionDate: '2023-01-10' }
];

  { id: 6, severity: 'Major', detectionDate: '2023-01-15', resolutionDate: '2023-01-18' },
  { id: 7, severity: 'Minor', detectionDate: '2023-01-16', resolutionDate: '2023-01-17' }

Coverage Metrics

  • Code Coverage - Percentage of code executed during testing
  • Requirements Coverage - Percentage of requirements covered by tests
  • Test Case Coverage - Percentage of test cases executed
  • Risk Coverage - Percentage of identified risks covered by tests

Interactive Diagram: Types of Code Coverage

Statement Coverage
Statement Coverage

Measures the percentage of statements executed during testing

function example(a, b) {
  let result = 0;
  if (a > 0) {
    result = a + b;  // Statement 1
  } else {
    result = a - b;  // Statement 2
  }
  return result;     // Statement 3
}

If we test with a = 1, b = 2, we execute statements 1 and 3, giving 67% statement coverage.

Branch Coverage
Branch Coverage

Measures the percentage of decision branches executed during testing

function example(a, b) {
  let result = 0;
  if (a > 0) {       // Branch 1: a > 0
    result = a + b;
  } else {           // Branch 2: a <= 0
    result = a - b;
  }
  return result;
}

If we test with a = 1, b = 2, we execute Branch 1 but not Branch 2, giving 50% branch coverage.

Path Coverage
Path Coverage

Measures the percentage of possible paths through the code executed during testing

function example(a, b) {
  let result = 0;
  if (a > 0) {
    if (b > 0) {     // Path 1: a > 0, b > 0
      result = a + b;
    } else {         // Path 2: a > 0, b <= 0
      result = a;
    }
  } else {
    if (b > 0) {     // Path 3: a <= 0, b > 0
      result = b;
    } else {         // Path 4: a <= 0, b <= 0
      result = 0;
    }
  }
  return result;
}

If we test with (a=1, b=2) and (a=-1, b=-2), we execute Paths 1 and 4, giving 50% path coverage.

Function Coverage
Function Coverage

Measures the percentage of functions called during testing

function add(a, b) {
  return a + b;
}

function subtract(a, b) {
  return a - b;
}

function multiply(a, b) {
  return a * b;
}

function calculate(a, b, operation) {
  if (operation === 'add') {
    return add(a, b);
  } else if (operation === 'subtract') {
    return subtract(a, b);
  } else if (operation === 'multiply') {
    return multiply(a, b);
  }
  return 0;
}

If we test with operation = 'add', we call add() and calculate(), giving 50% function coverage (2 out of 4 functions).

Test Execution Metrics

  • Test Pass Rate - Percentage of tests that pass
  • Test Execution Time - Time taken to execute tests
  • Test Progress - Percentage of planned tests executed
  • Test Effectiveness - Number of defects found per test case
  • Test Efficiency - Number of defects found per testing hour
  • Automation Coverage - Percentage of tests that are automated
Example: Generating Test Execution Metrics Report
// JavaScript function to generate test execution metrics
function generateTestExecutionReport(testResults, plannedTests, testingHours) {
  // Test Pass Rate = (Passed Tests / Total Tests) * 100
  const totalTests = testResults.length;
  const passedTests = testResults.filter(test => test.status === 'Pass').length;
  const passRate = (passedTests / totalTests) * 100;
  
  // Test Execution Time (average in seconds)
  const totalExecutionTime = testResults.reduce((sum, test) => sum + test.executionTime, 0);
  const avgExecutionTime = totalExecutionTime / totalTests;
  
  // Test Progress = (Executed Tests / Planned Tests) * 100
  const progress = (totalTests / plannedTests) * 100;
  
  // Test Effectiveness = Defects Found / Number of Test Cases
  const defectsFound = testResults.reduce((sum, test) => sum + test.defectsFound, 0);
  const testEffectiveness = defectsFound / totalTests;
  
  // Test Efficiency = Defects Found / Testing Hours
  const testEfficiency = defectsFound / testingHours;
  
  // Automation Coverage = (Automated Tests / Total Tests) * 100
  const automatedTests = testResults.filter(test => test.isAutomated).length;
  const automationCoverage = (automatedTests / totalTests) * 100;
  
  return {
    passRate,
    avgExecutionTime,
    progress,
    testEffectiveness,
    testEfficiency,
    automationCoverage,
    summary: {
      totalTests,
      passedTests,
      failedTests: totalTests - passedTests,
      defectsFound,
      automatedTests,
      manualTests: totalTests - automatedTests
    }
  };
}

// Example usage
const testResults = [
  { id: 1, name: 'Login Test', status: 'Pass', executionTime: 5.2, defectsFound: 0, isAutomated: true },
  { id: 2, name: 'Search Test', status: 'Fail', executionTime: 8.7, defectsFound: 2, isAutomated: true },
  { id: 3, name: 'Payment Test', status: 'Pass', executionTime: 12.3, defectsFound: 0, isAutomated: true },
  { id: 4, name: 'Profile Update Test', status: 'Pass', executionTime: 6.5, defectsFound: 0, isAutomated: false },
  { id: 5, name: 'Checkout Test', status: 'Fail', executionTime: 15.1, defectsFound: 3, isAutomated: false }
];

const plannedTests = 10; // Total planned tests
const testingHours = 8; // Hours spent testing

const report = generateTestExecutionReport(testResults, plannedTests, testingHours);
console.log(report);

Test Reporting

Test reporting involves communicating testing results and insights to stakeholders in a clear, concise, and actionable format.

Types of Test Reports

  • Test Summary Report - Overview of testing activities and results
  • Defect Report - Details of defects found during testing
  • Test Progress Report - Status of testing activities against the plan
  • Test Metrics Report - Quantitative measurements of testing
  • Test Coverage Report - Extent of test coverage achieved
  • Release Readiness Report - Assessment of whether the software is ready for release

Tailoring Reports for Different Stakeholders

Stakeholder Information Needs Report Focus
Executive Management Overall quality, risks, business impact High-level summary, key metrics, risk assessment
Project Manager Progress, issues, resource needs Test progress, blockers, resource utilization
Development Team Technical details, defect information Detailed defect reports, code coverage, technical metrics
Product Owner Feature readiness, user impact Feature-level test results, user-facing issues
QA Team Detailed test results, test effectiveness Comprehensive test metrics, test efficiency, coverage

Effective Reporting Practices

  • Be Clear and Concise - Focus on key information

Risk-Based Testing

Risk-based testing is an approach that prioritizes testing activities based on the level of risk associated with different aspects of the software.

Risk-Based Testing Process

  1. Identify Risks - Determine what could go wrong
  2. Analyze Risks - Assess probability and impact
  3. Prioritize Risks - Rank risks based on severity
  4. Mitigate Risks - Plan testing to address high-priority risks
  5. Monitor Risks - Track risk status throughout testing
Example: Risk Assessment Matrix
// JavaScript function to calculate risk priority
function calculateRiskPriority(risks) {
  // Risk priority = Probability * Impact
  const riskMatrix = risks.map(risk => {
    const priority = risk.probability * risk.impact;
    let level;
    
    if (priority >= 15) {
      level = 'Critical';
    } else if (priority >= 8) {
      level = 'High';
    } else if (priority >= 3) {
      level = 'Medium';
    } else {
      level = 'Low';
    }
    
    return {
      ...risk,
      priority,
      level
    };
  });
  
  // Sort risks by priority (highest first)
  return riskMatrix.sort((a, b) => b.priority - a.priority);
}

// Example usage
const risks = [
  { id: 1, description: 'Payment processing failure', probability: 3, impact: 5 },
  { id: 2, description: 'User data exposure', probability: 2, impact: 5 },
  { id: 3, description: 'Performance degradation under load', probability: 4, impact: 3 },
  { id: 4, description: 'UI rendering issues on mobile devices', probability: 4, impact: 2 },
  { id: 5, description: 'Incorrect calculation of order totals', probability: 2, impact: 4 }
];

const prioritizedRisks = calculateRiskPriority(risks);
console.log(prioritizedRisks);
